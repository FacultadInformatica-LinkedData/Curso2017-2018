Analisis para el dataset 'estacion-bicicleta.csv'

Este dataset contiene información sobre las estaciones de bici disponibles en Zaragoza
Hay en total 50 datos con los siguientes campos:

  • • id: Es un enlace para mostrar la información de la estación con ese id. -- > Tipo de datos: URI.
  • • about: Muestra más información de la estación. -- > Tipo de datos: URI.
  • • title: Muestra el nombre de la calle donde se encuentra la estacion. -- > Tipo de datos: String.
  • • estado: Muestra si la estacion esta abierta -- > Tipo de datos: String.
  • • bicisDisponibles: Muestra el numero de bicicletas disponibles en la estacion. -- > Tipo de datos: Entero.
  • • anclajesDisponibles: Muestra el numero de espacios dispoibles para dejar las bicicletas. -- > Tipo de datos: Entero.
  • • geometry: Muestra las coordenadas de la estacion -- > Tipo de datos: Entero.
  • • lastUpdate: Muestra la fecha de la última actualizacion de la informacion. -- > Tipo de datos: xsd:dateTime .
  • • description: Muestra un resumen de la informacion disponible -- > Tipo de datos: String.
  • • icon: Muestra un enlace al icono de una estacion de bicicletas. -- > Tipo de datos: URI

    • Analisis para el dataset 'poste-autobus.csv'

    Este dataset contiene un listado de las marquesinas de autobús de la ciudad de Zaragoza.
    Hay en total 1208 marquesinas con los siguientes campos:

  • • id: Es el identificador de la marquesina en cuestión. (Enlace a un xml). -- > Tipo de datos: URI.
  • • title: Muestra el número de la marquesina, el destino y las líneas de autobus que pasan por ella. -- > Tipo de datos: String.
  • • geometry: Muestra las coordenadas de una marquesina. -- > Tipo de datos: float
  • • link: Muestra un enlace para el visionado en web de la información de la marquesina. -- > Tipo de datos: URI
  • • icon: Muestra un enlace al icono de un autobus. -- > Tipo de datos: URI

    • Resource Naming Strategy

    A continuación, se detalla la estrategia de nombrado de recursos

    • Dado que no se espera tener una gran cantidad de datos, utilizaremos hash URIS.
    • El dominio de las URI será www.zaramove.es
    • El path de las URI será el siguiente: http://www.zaramove.es/transportes
    • Los pattern serán los siguientes: Para las clases en la ontologia http://www.zaramove.es/transportes/ontology/#. Para las propiedades en la ontología http://www.zaramove.es/transportes/ontology/#. Y para las instancias será en la ontología http://www.zaramove.es/transportes/resource/#.
